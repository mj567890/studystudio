**日期**：2026-04-29 ~ 2026-04-30
**状态**：已完成并部署

---

## 1. 动机

教师偏好（教学指导）原为自由文本输入，一次性使用，无复用机制。教师精调后置，AI 先按默认风格生成完整课程，教师不满再精调——浪费算力。

**核心思路**：模板 = 命名的、可复用的 `teacher_instruction` 文本。选择模板即预填教学指令，经现有 `TEACHER_INSTRUCTION_PREFIX` 包装后注入全部 4 个 LLM Prompt。

### 2026-04-30 重大更新：三课型子模板

每个模板现包含三个段落，分别定义三种课型的教学风格：

```
[原理课] → theory（概念定义、原理推导、术语辨析）
[任务课] → task（分步操作、工具使用、技能练习）
[实战课] → project（综合项目、真实场景、完整产出）
```

生成流程：
1. `CLUSTER_CHAPTER_PROMPT` 新增 `chapter_type` 字段判定课型
2. `_parse_template_sections()` 解析模板为三段子模板
3. `_gen_content()` 根据 `chapter_type` 选择对应子模板
4. 子模板经 `TEACHER_INSTRUCTION_PREFIX` 注入 `CHAPTER_CONTENT_PROMPT`

阶段（foundation/practice/assessment）决定难度和综合性，不影响模板选择。
课型（theory/task/project）决定用什么教学风格，与阶段正交。

理论基础：Merrill CDT (Concepts/Procedures/Principles) + Gagné 智慧技能层级 + VET 行动导向

## 5. 关键设计决策

1. **模板即文本**：不建独立的 Prompt 体系，模板内容作为 `teacher_instruction` 字符串沿现有链路流通（KISS）
2. **组件复用**：TemplateSelector 同时用于 UploadView（上传自动生成）和 RepairPathView（手动生成）（DRY）
3. **向后兼容**：不选模板 = 空 instruction = 默认风格 = 现有行为不变；旧模板无标记则三种课型共用同一内容
4. **Pipeline 修改**：`blueprint_tasks.py` 新增 `_parse_template_sections()` + `CLUSTER_CHAPTER_PROMPT` 课型判定 + `_gen_content()` 按课型选子模板
5. **课型=教学方式分类，阶段=难度分级**：二维正交，不混淆

## 6. 三课型模板结构

5 个系统模板均以三段标记格式存储：

```
[原理课]
正文结构："概念定义 → 底层原理 → 为什么重要 → 与其他概念的关系"。
术语首次出现用【术语名】标注。误区纠正侧重概念层面的常见混淆。
代码示例用最小可验证片段。自适应扩展补充学术前沿或工业实践差异。

[任务课]
正文结构："任务目标 → 所需工具/环境 → 分步操作 → 预期结果 → 验证方法"。
每一步给出可直接复制运行的命令。误区纠正侧重新手常见操作错误。
代码示例提供完整可运行的工具级代码。

[实战课]
正文结构："项目背景 → 需求拆解 → 方案设计 → 实施步骤 → 验收标准 → 复盘反思"。
代码示例为项目级完整代码或关键模块。误区纠正侧重架构/设计层面的失误。
```

每个系统模板的三段内容侧重不同（系统默认=均衡，实操导向=偏操作，理论基础=偏推导，速成精简=偏结论）。

前端编辑表单显示三个带课型标签的 textarea，查看对话框分三段展示。新建模板时预填系统默认的三段内容。

## 7. 已修复 Bug

**问题**：TemplateSelector 下拉框、管理对话框、新建表单全部为空。

**根因**：axios 响应拦截器 `res => res.data` 已去掉 Axios 外层信封，但组件中对 API 响应体又多解了一层 `.data`。

API 返回：
```json
{"code": 200, "data": {"templates": [...]}}
```

拦截器后组件拿到的是 `{code, data: {templates: [...]}}`，正确访问为 `res.data.templates`。

错误代码：
```typescript
templates.value = res.data.data.templates || []  // .data 多了一层
templates.value.push(res.data.data)               // 同上
templates.value[idx] = res.data.data              // 同上
```

修复后：
```typescript
templates.value = res.data.templates || []
templates.value.push(res.data)
templates.value[idx] = res.data
```

## 8. 验证方法

1. 打开上传页面 → 课程模板下拉 → 应显示 5 个系统模板
2. 点击 ☰ 管理模板 → 点击"查看" → 应显示三课型分段内容
3. 点击新建模板 → 表单应预填三课型内容（原理课/任务课/实战课分别显示）
4. 选择模板后上传文件 → 生成的课程应按课型匹配对应子模板
5. 刷新或重新生成 → 不同性质的章节应体现不同的教学侧重
