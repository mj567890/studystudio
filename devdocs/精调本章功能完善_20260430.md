# 精调本章功能完善

**日期**：2026-04-30
**状态**：已部署（本地 + 服务器）

---

## 1. 改动背景

原"精调本章"存在 5 个问题：
1. 代码重复 — KnowledgeView / TutorialView 各 25 行相同逻辑
2. 无备份回滚 — 改坏无法恢复
3. 无模板复用 — 每次手写指令
4. 无快捷引导 — 纯 textarea，不熟悉 AI 的用户不知道写什么
5. 历史不可见 — 后端已追踪 version，前端不展示

## 2. 改动清单

### 2.1 数据库

**migrations/027_teacher_refinement.sql** 追加：
```sql
ALTER TABLE skill_chapters ADD COLUMN IF NOT EXISTS previous_content_text text;
```

### 2.2 后端（router.py）

| 端点 | 方法 | 说明 |
|------|------|------|
| `/admin/courses/chapters/{id}` | GET | 章节详情（版本号 + 时间 + 是否有历史版本） |
| `/admin/courses/chapters/{id}/refine` | POST | 增强：精调前自动备份 content_text → previous_content_text |
| `/admin/courses/chapters/{id}/rollback` | POST | 新增：previous_content_text → content_text，版本号 -1 |

章节列表 `/admin/courses/{bid}/chapters` 返回增加 `refinement_version` / `refined_at` 字段。

### 2.3 前端

| 文件 | 操作 | 说明 |
|------|------|------|
| `composables/useRefineChapter.ts` | 新建 | 共享精调逻辑（打开/关闭/提交/回滚） |
| `components/RefineChapterDialog.vue` | 新建 | 增强对话框 |
| `views/admin/KnowledgeView.vue` | 修改 | 删除重复代码，使用 composable + 组件 |
| `views/tutorial/TutorialView.vue` | 修改 | 同上 |
| `api/index.ts` | 修改 | +rollbackChapter +getChapterDetail |

**对话框功能**：
- 版本信息（版本 N + 上次精调时间）
- 套用模板下拉框（加载已有模板，选中填入 textarea）
- 10 行 textarea + 5 个快捷指令按钮
- 测验/讨论题重生成 checkbox
- 回滚到上一版本按钮（有历史版本时显示）
- 对话框宽度 720px

## 3. 设计原则

- **KISS**：备份只存最近一版（单列），不建完整历史表
- **DRY**：useRefineChapter + RefineChapterDialog 消除两处 ~50 行重复代码
- **防御性**：备份在前、覆盖在后
- **向后兼容**：previous_content_text 为 NULL 时回滚按钮不可见
