## 1. 动机

教师偏好（教学指导）原为自由文本输入，一次性使用，无复用机制。教师精调后置，AI 先按默认风格生成完整课程，教师不满再精调——浪费算力。

**核心思路**：模板 = 命名的、可复用的 `teacher_instruction` 文本。选择模板即预填教学指令，经现有 `TEACHER_INSTRUCTION_PREFIX` 包装后注入全部 4 个 LLM Prompt。

```
├──────────────────────────────────────┤
│  API: /api/course-templates          │
│  6 端点 (CRUD + 设默认)              │
├──────────────────────────────────────┤
│  前端: TemplateSelector.vue（复用）   │
│  前端: TemplateManageView.vue（独立页）│
├──────────────────────────────────────┤
│  Pipeline: on_knowledge_extracted    │
│  → 查 3 个默认模板                    │
│  → synthesize_blueprint()            │
│  → _gen_content() 按 chapter_type    │
│    选择对应模板注入 Prompt            │
└──────────────────────────────────────┘
```

### 2.1 三课型独立模板体系（当前实现 v2.8）

LLM 在聚类阶段（`CLUSTER_CHAPTER_PROMPT`）判定每个章节的 `chapter_type`（theory/task/project），生成时根据类型从对应模板取指令注入 Prompt：

```
chapter_type = "theory"  → 使用"原理课"模板（默认：理论基础）
chapter_type = "task"    → 使用"任务课"模板（默认：实操导向）
chapter_type = "project" → 使用"实战课"模板（默认：系统默认）
```

- **每个课型对应一个完整独立模板**，不分割模板内部结构
- `knowledge_spaces` 表存储 4 个 template_id 引用（1 个 legacy + 3 个课型专用）
- 上传时自动保存模板选择到空间，自动生成时从空间读取
- 手动生成路径（RepairPathView）也支持三课型选择，实时传入

**理论基础**：Merrill CDT (Concepts/Procedures/Principles) + Gagné 智慧技能层级 + VET 行动导向教学（详见 `课程模板来源与理论基础_20260430.md`）

### 2.2 历史：三段式方案（已回滚 2026-04-30）

> **此方案已废弃**，详见备份文件和 migration 032。核心问题：把每个模板硬拆成 `[原理课]`/`[任务课]`/`[实战课]` 三个段落，导致模板内容臃肿、用户编辑困难、语义割裂。正确方案是 5 个独立模板 × 3 个课型选择器，每个课型选不同的完整模板。

## 5. 关键设计决策

1. **模板即文本**：不建独立的 Prompt 体系，模板内容作为 `teacher_instruction` 字符串沿现有链路流通（KISS）
2. **组件复用**：TemplateSelector 同时用于 UploadView（上传自动生成）和 RepairPathView（手动生成）（DRY）
3. **向后兼容**：不选模板 = 空 instruction = 默认风格 = 现有行为不变
4. **独立模板 × 课型选择器**：3 个课型选择器各选不同模板，不把模板拆成三段（v2 最终方案）
5. **课型 = 教学方式分类，阶段 = 难度分级**：二维正交，不混淆
6. **"+ " 按钮预填**：TemplateSelector 的 "+" 按钮以当前选中模板为基础预填内容，模板管理页的 "+" 为空白表单
7. **系统模板保护**：is_system 模板不可编辑/删除（403）

## 7. TemplateSelector 组件规范

### Props
```typescript
{
  modelValue?: string | null        // 当前选中的 template_id
  placeholder?: string              // 下拉框占位文本
  defaultTemplateName?: string      // 加载后自动选中的模板名称
}
```

### Emits
```typescript
{
  'update:modelValue': [value: string | null]  // 选中 template_id 变化
  'select': [content: string]                   // 选中模板的 content 文本
}
```

### 使用示例（RepairPathView 三课型）
```html
<!-- 原理课 -->
<TemplateSelector
  v-model="selectedTheoryTemplateId"
  placeholder="概念/原理类"
  default-template-name="理论基础"
  @select="ct => theoryContent = ct"
/>
<!-- 任务课 -->
<TemplateSelector
  v-model="selectedTaskTemplateId"
  placeholder="工具/操作类"
  default-template-name="实操导向"
  @select="ct => taskContent = ct"
/>
<!-- 实战课 -->
<TemplateSelector
  v-model="selectedProjectTemplateId"
  placeholder="项目/综合类"
  default-template-name="系统默认"
  @select="ct => projectContent = ct"
/>
```

## 8. TemplateManageView（独立管理页）

路由：`/templates`，菜单：资源 → 课程模板

### 功能
- **表格**：名称（含系统标签）、教学指令预览（前100字截断）、操作列
- **新建模板**（"+ 新建模板"）→ 空白表单
- **查看**（"查看"）→ 弹窗显示完整模板内容
- **以此为基础新建**（"以此为基础新建"）→ 复制该模板 content 到新建表单
- **系统模板**：标记"系统"标签，仅有"查看"和"以此为基础新建"，无编辑/删除

### 与 TemplateSelector 的区别
| 场景 | TemplateSelector "+" | 管理页 "新建" |
|------|---------------------|-------------|
| 初始内容 | 预填当前选中模板 | 空白 |
| 目的 | 快速基于现有模板微调 | 从零创建全新模板 |

## 9. 后端 Pipeline 细节

### `blueprint_tasks.py` 关键代码路径

```python
# synthesize_blueprint 签名
def synthesize_blueprint(topic_key, space_id, force=False,
                         teacher_instruction=None,
                         type_instructions=None):  # 新增
    ...

# _gen_content 中按课型选择模板
type_instruction_blocks = {}
if type_instructions:
    for ct in ("theory", "task", "project"):
        content = type_instructions.get(ct, "")
        type_instruction_blocks[ct] = TEACHER_INSTRUCTION_PREFIX.format(
            instruction=content
        ) if content else ""

chapter_type = meta.get("chapter_type", "task")
selected_instruction = type_instruction_blocks.get(chapter_type) or instruction_block
```

### `main.py` on_knowledge_extracted 处理器

```python
# 从 knowledge_spaces 读取 3 个课型模板引用
# LEFT JOIN course_templates 获取 content
# 组装 type_instructions dict 传入 synthesize_blueprint
type_instructions = {
    "theory": theory_template_content,
    "task": task_template_content,
    "project": project_template_content,
}
```

## 10. 已修复 Bug

### Bug 1：TemplateSelector 数据为空

**问题**：下拉框、管理对话框、新建表单全部为空。

**根因**：axios 响应拦截器 `res => res.data` 已去掉 Axios 外层信封，但组件中又多解了一层 `.data`。

**修复**（TemplateSelector.vue 4 处）：
```typescript
// 错误（修复前）
templates.value = res.data.data.templates || []
templates.value.push(res.data.data)
templates.value[idx] = res.data.data

// 正确（修复后）
templates.value = res.data.templates || []
templates.value.push(res.data)
templates.value[idx] = res.data
```

### Bug 2：三课型设计走偏（架构级回滚）

**问题**：将每个模板硬拆为 `[原理课]`/`[任务课]`/`[实战课]` 三段，导致模板臃肿、用户编辑困难。

**修复**：migration 032 回滚 → 恢复 5 个完整模板 → 改为 3 个 TemplateSelector 各选不同模板。

### Bug 3：SystemHealthView 双重 data 解包（同根因）

**问题**：系统监控页数据库/RabbitMQ 标签始终显示"异常"，实际 API 返回正常。

**根因**：与 Bug 1 相同 —— axios 拦截器 `res => res.data` + `const { data } = ...` 解构后再接 `.data` 多解一层。

**详细记录**：见 `SystemHealthView双重data解包修复_20260430.md`

## 11. 验证方法

1. 打开上传页面 → 三个课型模板下拉框各出现，原理课默认"理论基础"，任务课默认"实操导向"，实战课默认"系统默认"
2. TemplateSelector "+" 按钮 → 预填当前选中模板内容
3. 菜单 → 资源 → 课程模板 → 进入管理页 → 查看/以此为基础新建
4. 管理页 "新建" → 空白表单；管理页 "以此为基础新建" → 复制对应模板内容
5. 选择模板后上传文件 → 自动生成 → 不同课型章节体现不同教学侧重
6. 在 RepairPathView 选择不同课型模板 → 填写额外教学指导 → 生成 → 验证模板效果
7. 不选模板直接生成 → 现有默认行为不变
8. 尝试编辑/删除系统模板 → 应返回 403
