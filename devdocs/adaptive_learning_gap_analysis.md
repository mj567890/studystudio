# 自适应学习路径 — 差距分析与完善路线

**日期：** 2026-05-02
**状态：** 待决策

---

## 一、现状总览

v2.2 中自适应学习路径的代码框架完整（4 个 Service、7 个 API 端点、4 个前端页面），但核心算法停留在"固定阈值 + 固定增量 + 图遍历"层面，不具备真正的自适应能力。

**根本缺失：没有学生模型（Student Model）。** 系统用一个标量 `mastery_score` (float 0-1) 表示学生对知识点的掌握程度，没有概率推断、没有不确定性量化、没有根据答题表现动态调整参数的能力。

---

## 二、逐项差距

### 2.1 分级测验冷启动

| 维度 | 当前 | 理想 |
|------|------|------|
| 题目来源 | LLM 预生成（7 天 TTL），降级为自评 | IRT 校准题库 + CAT 自适应选题 |
| 评分方式 | 每领域 2 题 → 四档查表映射 | 基于答题模式推断能力参数 θ |
| 降级方案 | "你觉得自己懂不懂 X？"（4 档自评） | 基于材料难度的先验知识估计 |
| 领域内覆盖 | 一个分数赋给领域内所有 entity | 逐 entity 的后验概率 |
| 冷启动延迟 | 依赖 LLM 预生成任务完成 | 基于知识图谱结构即可运行 |

**差距：** 没有 IRT 模型，无法区分"题目太难答不对"和"学生太弱答不对"。

### 2.2 漏洞扫描

| 维度 | 当前 | 理想 |
|------|------|------|
| 分类逻辑 | 两个固定阈值（0.5 / 0.8） | 贝叶斯后验概率 < 某置信度 |
| Gap 类型 | 全用阈值区间硬映射（<0.5→DEFINITION） | 基于错误模式推断真实缺口类型 |
| 难度感知 | 无 | 区分"因为难而不会"和"因为基础缺而不会" |
| 优先级 | fixed: 0.2/0.5/0.8 三档 | 动态：失误代价 × 依赖权重 × 遗忘风险 |

**差距：** 所有 < 0.5 全标为 `DEFINITION`，但代码里已经定义了 6 种 `GapType` 枚举——有架构意识，无实现。

### 2.3 补洞路径

| 维度 | 当前 | 理想 |
|------|------|------|
| 生成算法 | 拓扑排序（Kahn）+ 硬截断 20 步 | 带约束的最优排序问题 |
| 排序依据 | 仅依赖深度 | 依赖深度 + 难度 + 学生历史学习速率 + 时间预算 |
| 个人化 | 无 | 每学生不同路径（同一 topic 不同人走不同路线） |
| 时间估算 | 章节内容长度/500 | 学生历史阅读速度 |
| 中途调整 | 不支持 | 学完一章后根据最新掌握度重新规划 |

**差距：** 本质是一个图遍历工具，不是学习路径规划器。所有同 topic 学生拿到的路径几乎一样。

### 2.4 掌握度更新

| 维度 | 当前 | 理想 |
|------|------|------|
| 更新方式 | 固定增量：+0.2（correct）/ -0.1（wrong） | 贝叶斯更新：P(knows | evidence) |
| 难度区分 | 无——答对难题和答对简单题一样 | 难题答对 → 更大后验提升 |
| 遗忘建模 | 固定 λ=0.1 指数衰减，从不更新 λ | 每学生/每 entity 的个性化遗忘率 |
| 更新触发 | 仅在章节完成/测验提交时 | 任何学习行为（阅读、讨论、提问）都应更新 |
| 上限保护 | 硬上限 0.9 | 概率天然有上限，不需要硬编码 |

**差距：** 这是整个系统最弱的一环。固定 delta 意味着：一个完全掌握的学生（真实概率 100%）和完全不懂的学生（真实概率 0%），答对同一题的"增益"都是 +0.2。

### 2.5 数据采集了但没用

| 数据 | 位置 | 现状 |
|------|------|------|
| `chapter_quiz_attempts` | 015 迁移 | 写入 + 只读 API 展示，不反馈到模型 |
| `chapter_reflections` | 015 迁移 | 学生自评 + AI 反馈，不进入任何计算 |
| `tutorial_annotations` | 教程标注 | 展示用，不进补洞路径 |
| `error_patterns` | eight_dim | API 返回 top-10 薄弱 entity，但 GapScan 不消费 |

## 三、理想学生模型

如果要做"对"，核心是引入一个**贝叶斯知识追踪（BKT, Bayesian Knowledge Tracing）**模型。

```
每个 (student, entity) 维护四个参数：
  P(L₀)   — 初始掌握概率（先验）
  P(T)    — 从不会到会的学习概率（transition）
  P(G)    — 不会但猜对的概率（guess）
  P(S)    — 会但答错的概率（slip）

每次答题后，贝叶斯更新：
  P(learned | correct)   = P(L) * (1-P(S)) / P(correct)
  P(learned | incorrect) = P(L) * P(S) / P(incorrect)

最终输出：P(L) ∈ [0, 1]，带有明确的不确定性。
```

**为什么 BKT 而不是 transformer-based DKT：**
- DKT 需要大量训练数据（至少几千学生），我们目前没有
- BKT 参数可解释（教师能看懂为什么系统认为学生不会）
- BKT 冷启动友好（可以用材料难度和知识图谱结构估计先验）
- BKT 更新快（每次答题 O(1)），不需要 GPU

## 四、路线选项

### 选项 A：集中打磨课程生成（v2.3 不改自适应）

**做法：** v2.3 继续投入阶段性经验校准、case/project 课型、材料版本冲突检测。自适应学习维持现状，不加 BKT。

**优点：** 专注差异化优势，课程生成是真正的护城河
**缺点：** README 里"自适应学习"继续名不副实，学生端体验无提升
**工时：** 0（不做）
**风险：** 如果竞品推出真正自适应的学生端，我们被动

### 选项 B：轻量增强（不改模型，修阈值）

**做法：**
1. Gap 类型从错误模式推断（用 `chapter_quiz_attempts.wrong_entity_ids` 反推真实缺口）
2. 补洞路径加入难度权重和去重优化
3. 遗忘率 λ 按 entity 的难度级别分档（core/concept/advanced 不同 λ）
4. 利用已采集但未用的数据（reflections、annotations）做简单的启发式优化

**优点：** 工时小（约 2-3 天），不改数据模型
**缺点：** 仍是阈值驱动，根本上不解决"没有学生模型"的问题
**工时：** ~3 人天
**风险：** 改完还是只能算"基础版+"，只是更好看一点

### 选项 C：引入 BKT 学生模型（v2.3 核心）

**做法：**
1. `learner_knowledge_states` 从单 scalar 扩展为 BKT 四参数
2. 冷启动：基于 placement quiz 的答题模式（而非自评）推断 P(L₀)
3. 每次答题 → 贝叶斯更新 P(L)
4. 漏洞 = P(L) < 0.5 且置信区间窄（确定不会），而非 score < 0.5
5. 补洞路径：在拓扑排序基础上，用 P(L) 做优先权重
6. 遗忘率 λ 随学生历史学习速率自适应调整

**优点：** 真正有"自适应"内核，学生端体验质变，可以和课程生成形成双引擎
**缺点：** 工时大，需改数据模型 + 需 migration
**工时：** ~8-12 人天
**风险：** P(G) 和 P(S) 需要数据积累才能校准，前几个月可能不准

### 选项 D：分两步走（B + 为 C 做准备）

**v2.3（现在）：** 轻量增强 + 数据采集
- 做选项 B 的 4 项改进
- 建 `content_effectiveness` 表（原计划 migration 037）
- 给 `learner_knowledge_states` 加 `p_learned`、`guess_prob`、`slip_prob` 字段（先写 0，为 BKT 预留）
- 把 quiz_attempts、reflections、annotations 的数据接入 gap 推断

**v2.4（下个版本）：** 引入 BKT
- 开启 `p_learned` 的贝叶斯更新
- 用积累的答题数据校准 P(G) 和 P(S)
- 替换所有 threshold-based 逻辑为 probability-based

**优点：** 渐进式，每个版本都有可见改进，数据积累为 BKT 做准备
**缺点：** 跨越两个版本
**工时：** v2.3 ~3 人天 / v2.4 ~8 人天
**风险：** 低——每步都有 fallback

---

## 五、建议

**推荐选项 D。**

理由：
1. 课程生成仍然是 v2.3 主线（case/project 课型、材料版本冲突），不宜分兵太多
2. 但自适应学习不能一直停在"基础版"——数据已经采集了，不做任何利用是说不过去的
3. 先做轻量增强（利用现有数据 + 预留 BKT 字段），下个版本引入真正的概率模型
4. 两个版本后，课程生成 + 自适应学习形成双引擎——这是竞品难以复制的
