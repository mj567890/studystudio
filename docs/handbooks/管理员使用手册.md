# 自适应学习平台 管理员使用手册
**版本：1.0.0 | 适用对象：系统管理员、知识审核员、教师**

## 第一章 管理端入口与权限

### 1.1 访问管理后台

管理后台地址：`https://your-domain.com/admin`

不同角色的可用功能：

| 功能模块 | 系统管理员 | 知识审核员 | 教师 |
|---|---|---|---|
| 用户管理 | ✅ | ❌ | ❌ |
| 全局知识库审核 | ✅ | ✅ | ❌ |
| 课程知识库管理 | ✅ | ❌ | ✅（本课程） |
| 教程内容审核 | ✅ | ✅ | ✅（本课程） |
| 学生学习数据 | ✅ | ❌ | ✅（本课程） |
| 系统配置 | ✅ | ❌ | ❌ |
| 事件队列监控 | ✅ | ❌ | ❌ |

## 第二章 知识库管理

### 2.1 知识空间说明

平台采用三层知识空间，审核策略不同：

| 空间 | 内容来源 | 审核要求 | 可见范围 |
|---|---|---|---|
| 全局知识库 | 管理员/知识审核员上传 | 必须人工审核通过后才能被检索 | 所有用户 |
| 课程知识库 | 教师上传 | 教师可自审，建议抽查 | 该课程学员 |
| 个人知识库 | 学习者上传 | 无需审核（个人可见） | 仅本人 |

### 2.2 审核待处理的知识点

**路径**：管理后台 → 知识审核 → 待审核列表

页面展示所有 `review_status = 'pending'` 的知识点，按创建时间排序。

**审核操作**：

| 操作 | 含义 | 说明 |
|---|---|---|
| 通过 | `review_status → approved` | 知识点进入正式知识库，可被检索 |
| 驳回 | `review_status → rejected` | 附上驳回原因，上传者可收到通知 |
| 编辑后通过 | 修改后通过 | 可直接修改 `canonical_name`、`short_definition` 等字段后通过 |

**审核建议**：
- 优先审核 `is_core = true` 的候选知识点（核心知识点影响所有学习者的路径推荐）
- 检查 `canonical_name` 是否规范（全称，无歧义）
- 检查 `domain_tag` 分类是否正确

### 2.3 处理归一化冲突

**路径**：管理后台 → 知识审核 → 疑似重复列表

当两个知识点的向量相似度 > 0.94（同层阈值），系统会推送到此列表供人工决策。

**处理方式**：

| 操作 | 适用场景 |
|---|---|
| 确认合并 | 两个知识点确实是同一概念（如"SQL注入"和"SQL注射"）→ 保留 `canonical_name` 更规范的一方 |
| 确认保留 | 两个知识点虽然相似但有实质差异 → 两者都保留，可添加 `related` 关系 |
| 标记为别名 | 一个是另一个的简称/别称 → 将简称加入 `aliases` 字段 |

### 2.4 上传文档到全局知识库

1. 进入「知识管理」→「上传文档」
2. 选择文件（PDF/Word/Markdown，最大 100MB）
3. 选择 `space_type = global`
4. 填写 `domain_tag`（如 `php-security`、`network-protocol`）
5. 提交后系统自动解析，完成后在「待审核」列表处理抽取结果

> **提示**：上传前建议先确认文档的 `domain_tag`，错误的分类会导致知识点无法被正确检索。

### 2.5 知识图谱健康度监控

**路径**：管理后台 → 知识图谱 → 健康报告

| 指标 | 正常范围 | 告警阈值 | 处理方式 |
|---|---|---|---|
| 孤岛率（无关系节点占比）| < 10% | > 15% | 检查孤立节点，手动添加关系或标记核心节点 |
| 弱连通分量数 | ≤ 3 | > 5 | 检查是否存在孤立的知识子图 |
| 核心节点覆盖率 | > 30% | < 30% | 将重要知识点的 `is_core` 标记为 true |

## 第三章 教程内容管理

### 3.1 查看待审核教程章节

**路径**：管理后台 → 内容审核 → 教程章节列表

系统自动生成的教程内容在质量评分不达标时会进入审核队列，显示具体未通过的维度：

| 质量维度 | 未通过的常见原因 |
|---|---|
| 章节覆盖完整性 | 正文未用 `{{知识点名称}}` 格式标注目标知识点 |
| 前置知识引用率 | 正文提及前置知识点但未使用 `{{}}` 格式标注 |
| 内容长度 | 内容过短（< 200 token）或过长（> 2000 token） |
| 逻辑连贯性 | 相邻段落话题切换过大（embedding 相似度低） |

**处理选项**：
- **人工编辑**：直接修改章节内容后标记通过
- **重新生成**：丢弃当前内容，重新触发 LLM 生成（消耗 Token）
- **强制通过**：内容质量可接受但评分偏低时（如专业术语密集导致连贯性分低）

### 3.2 配置教程质量阈值

**路径**：管理后台 → 系统配置 → 教程质量设置

> 注意：阈值需根据实际生成内容校准，初始值为保守设置。

| 配置项 | 初始值 | 说明 |
|---|---|---|
| 连贯性 embedding 阈值 | 0.4 | 建议上线后收集 100 篇内容的分布后调整 |
| 前置知识引用率最低要求 | 0.3 | 每章至少引用 30% 的前置知识点 |
| 最小 Token 数 | 200 | 低于此值视为内容过短 |
| 最大 Token 数 | 2000 | 超过此值视为内容过长 |

## 第四章 学习数据管理（教师视角）

### 4.1 查看班级学习进度

**路径**：管理后台 → 学习数据 → 班级概览

展示内容：
- 各学员当前掌握度热力图
- 全班薄弱点 Top 10（出现频率最高的薄弱知识点）
- 近 7 天活跃学习者数量
- 平均学习路径完成率

### 4.2 查看单个学生画像

**路径**：管理后台 → 学习数据 → 学员详情

可查看：
- 各知识点掌握度历史曲线
- 最近的对话诊断报告（障碍类型、错误推理模式）
- 已完成的教程章节列表
- 上传的个人学习资料

> **隐私说明**：教师只能查看本课程学员的数据，无法跨课程查看。个人上传资料内容不可见，仅可见数量和解析状态。

### 4.3 干预学习路径

当教师发现某学员的自动路径不合适时，可手动调整：

1. 进入学员详情 → 「学习路径」
2. 点击「调整路径」
3. 勾选或取消勾选知识点
4. 保存后，该学员下次查看路径时将使用调整后的版本

## 第五章 系统配置（仅系统管理员）

### 5.1 模型路由配置

**路径**：管理后台 → 系统配置 → 模型路由

| 路由 | 默认模型 | 说明 |
|---|---|---|
| teaching_chat_simple | gpt-4o-mini | 简单教学对话 |
| teaching_chat_complex | gpt-4o | 深度推理对话 |
| knowledge_extraction | gpt-4o | 知识抽取（高准确率优先）|
| quiz_generation | gpt-4o-mini | 题目生成 |

### 5.2 Token 预算管理

**路径**：管理后台 → 系统配置 → Token 预算

当日消耗量超过预算（默认 200 万）时，系统自动暂停低优先级批量任务（知识抽取、教程生成），保留实时对话服务。

**建议**：每两周查看实际消耗趋势，按实调整预算上限。

### 5.3 用户管理

**路径**：管理后台 → 用户管理

可执行操作：
- 重置用户密码
- 修改用户角色（learner / teacher / knowledge_reviewer / admin）
- 禁用/启用账号（`status = disabled/active`）
- 查看用户注册时间和最近活跃时间

### 5.4 死信队列处理

**路径**：管理后台 → 事件监控 → 死信队列

消费失败超过重试次数的事件会进入死信队列（DLQ），需人工处理：

| 处理方式 | 适用场景 |
|---|---|
| 重放 | 临时性故障（如 LLM 超时）已恢复，可重新处理 |
| 丢弃 | 事件已过时或对应数据已删除 |
| 查看详情 | 分析失败原因，排查代码 bug |

> **建议**：每天检查一次死信队列，积压超过 50 条时需及时处理。

## 第六章 常见问题

**Q：知识抽取准确率下降怎么办？**
A：进入「系统配置」→「Few-Shot 示例管理」，为相关领域补充高质量的示例（至少 3 组正例），重新触发该领域文档的知识抽取。

**Q：教程一直显示"生成中"不完成？**
A：查看 RabbitMQ 管理界面（`http://server:15672`），检查 `tutorial` 队列是否有积压。若 Celery Worker 重启后任务恢复处理，说明 Worker 之前崩溃；若任务在死信队列，查看具体错误。

**Q：某个学员的掌握度异常（全部为 0 或全部很高）？**
A：可能是冷启动时数据异常。进入学员详情 → 「重置画像」，让学员重新完成定位自检。

**Q：上传的文档显示"解析失败"？**
A：常见原因：① PDF 为扫描版（无文字层）；② 文档编码非 UTF-8；③ 文件超过 100MB。查看「文件详情」中的错误信息可获得具体原因。
