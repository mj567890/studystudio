## 第二章 快速启动（开发环境）

### 2.1 克隆仓库并配置环境变量

```bash
git clone <repo-url>
cd adaptive-learning-platform

# 复制并编辑环境变量文件
cp .env.example .env
# 必须填写：OPENAI_API_KEY=sk-...
```

### 2.2 启动所有服务

```bash
# 首次启动（构建镜像 + 启动）
docker-compose up --build -d

# 验证服务状态
docker-compose ps
```

**预期输出**（所有服务状态为 Up）：
```
NAME              STATUS    PORTS
api               Up        0.0.0.0:8000->8000/tcp
celery_worker     Up
celery_beat       Up
postgres          Up        0.0.0.0:5432->5432/tcp
redis             Up        0.0.0.0:6379->6379/tcp
rabbitmq          Up        0.0.0.0:5672->5672/tcp
minio             Up        0.0.0.0:9000->9000/tcp
```

### 2.3 验证服务健康

```bash
# API 健康检查
curl http://localhost:8000/health
# 期望返回：{"status": "ok", "version": "1.0.0", "env": "development"}

# 查看 API 文档
open http://localhost:8000/docs
```

### 2.4 初始化种子知识库（可选）

```bash
# 导入预置的核心知识点
docker-compose exec api python scripts/seed_knowledge.py

# 预生成冷启动题库（异步任务，后台执行）
docker-compose exec api python scripts/prebuild_banks.py
```

## 第三章 生产环境部署

### 3.1 使用生产配置文件

```bash
# 生产环境使用独立的 compose 文件（3副本 API + Nginx 负载均衡）
docker-compose -f docker-compose.prod.yml up -d
```

**docker-compose.prod.yml 关键配置**：
```yaml
services:
  nginx:
    image: nginx:alpine
    ports: ["80:80", "443:443"]
    volumes:
      - ./docker/nginx.conf:/etc/nginx/nginx.conf
      - ./docker/ssl:/etc/nginx/ssl

  api:
    deploy:
      replicas: 3   # 3副本满足100并发目标
    environment:
      - APP_DEBUG=false
      - JWT_SECRET_KEY=${JWT_SECRET_KEY}  # 使用强随机密钥
```

### 3.2 生产环境安全检查清单

- [ ] `JWT_SECRET_KEY` 已替换为强随机字符串（`openssl rand -hex 32`）
- [ ] `OPENAI_API_KEY` 通过 Docker secrets 或环境变量注入，未硬编码
- [ ] MinIO 默认密码已修改
- [ ] PostgreSQL 默认密码已修改
- [ ] RabbitMQ 默认密码已修改
- [ ] API 的 `APP_DEBUG=false`
- [ ] CORS 白名单已限制到实际域名
- [ ] 数据库已开启自动备份

## 第四章 数据库迁移

### 4.1 首次初始化

```bash
# 数据库初始化脚本在 PostgreSQL 容器启动时自动执行
# 对应文件：migrations/001_initial_schema.sql

# 验证表已创建
docker-compose exec postgres psql -U user -d adaptive_learning -c "\dt"
```

### 4.2 手动执行迁移

```bash
# 连接数据库
docker-compose exec postgres psql -U user -d adaptive_learning

# 执行迁移文件
\i /docker-entrypoint-initdb.d/001_schema.sql
```

### 4.3 Alembic 自动迁移（后续版本更新）

```bash
# 生成迁移文件
docker-compose exec api alembic revision --autogenerate -m "add_new_table"

# 执行迁移
docker-compose exec api alembic upgrade head

# 回滚一步
docker-compose exec api alembic downgrade -1
```

  -c "SELECT COUNT(*) FROM knowledge_entities WHERE embedding IS NOT NULL;"

## 第六章 故障排查

### 6.1 API 无法启动

```bash
# 检查数据库连接
docker-compose exec api python -c "
import asyncio
from apps.api.core.db import engine
asyncio.run(engine.connect())
print('DB OK')
"
```

### 6.2 pgvector 向量查询返回 bytes

**症状**：向量相似度查询返回原始字节而非浮点列表
**原因**：pgvector 编解码器未注册（C2 规范）
**修复**：确认 `core/db.py` 中 `on_connect` 事件监听器已注册

### 6.3 Celery 任务 RuntimeError: no running event loop

**症状**：`RuntimeError: There is no current event loop in thread`
**原因**：Celery 任务使用了顶层 `async def`（prefork 池不支持）
**修复**：确认所有任务体为同步函数，内部使用 `asyncio.run()`（C1 规范）

### 6.4 教学对话诊断写入偶发丢失

**症状**：BackgroundTask 中数据库写入失败
**原因**：使用了请求 session（已关闭）而非独立 session
**修复**：确认 `_run_diagnosis_update` 使用 `get_independent_db()`（D3+R1 规范）
