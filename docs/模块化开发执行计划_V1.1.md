## 五、关键技术约束（V2.6终审规范）

开发过程中所有工程师必须遵守以下规范，集成专员负责 CR 检查：

| 规范编号 | 约束内容 | 检查方式 |
|---|---|---|
| C1 | Celery任务体必须为同步函数，内部用`asyncio.run()`，禁止顶层`async def` | PR代码审查 |
| C2 | `engine.connect`事件中注册`register_vector`，禁止遗漏 | 集成测试：向量列返回`list[float]` |
| C3 | 所有MinIO操作通过`AsyncMinIOClient`，禁止直接调用boto3 | PR代码审查 |
| D1 | `generate_content`必须批量预取实体，禁止逐章单独查询 | 性能测试：N章节=1次DB查询 |
| D2 | `insert_if_not_exists`必须使用`xmax=0`技巧 | 并发测试：100并发不产生重复骨架 |
| D3+R1 | 诊断写入必须用`BackgroundTasks`+独立session，禁止`asyncio.create_task` | 代码审查 |
| B1 | `topological_sort_safe`全程使用`entity_id`字符串操作 | 单元测试：循环检测用例通过 |
| B2 | Redis锁必须Lua脚本原子释放 | 代码审查 |
| B3 | `PathStep`必须包含`dependency_depth`字段 | 接口联调：响应体字段验证 |
| R2 | `ingest`截断前必须记录`original_count` | 单元测试：截断计数验证 |
